Γενικά είχα περιορισμένο χρόνο, για να ασχοληθώ με την εργασία, επειδή κάνω την πρακτική
μου ταυτόχρονα και για αυτό ο Disease Aggregator είναι ελλειπής.

DISEASE AGGREGATOR:
    - Αρικά το πρόγραμμα διαβάζει τα δοσμένα arguments και δημιουργεί μια λίστα με
τα directories των χωρών.

    - Με round robin τακτική χωρίζει το πόσα dirs θα πάρει κάθε worker και στέλνει τα
ονόματα τους.

    - Πρωτόκολλο επικοινωνίας: Στέλνω αρχικά στον worker το πόσα dirs θα διαβάσει,
και έπειτα ένα ένα του στλένω τα ονόματα.

    - Ο worker μετά δημιουργεί ένα ένα τα paths των υπο-φακέλων και αποθηκεύει
τις εγγραφές σε μια λίστα που είναι Date sorted.

    - Δημιουργώ τις υπόλοιπες δομές μου και αποθηκεύω τις εγγραφές σε αυτές.

    * Ουσιαστικά έχω υλοποιήσει μέχρι το σημείο που ο κάθε worker, ετοιμάζει
τα Statistics για να τα στείλει στο parent process.

    * Η μόνη συνάρτηση που λειτουργεί είναι η /exit , η οποία όταν καλείται:
    - Κλείνει τις read/write επικοινωνίες.
    - Διαγράφει τα fifo αρχεία.
    - Σκοτώνει όλα τα child processes.
    - Απελευθερώνει όποια μνήμη έχουμε δεσμεύσει για τις δομές μας.


SCRIPT create-infiles.sh:
    - Το script είναι ολοκληρωμένο.

    - Κάνω όλους τους απαραίτητους ελέγχους για τα arguments που δίνονται.

    - Για να είναι λογικά τα αρχεία που φτιάχνω, υπάρχει μια πιθανότητα για τον
    τύπο της εγγραφής:

        - 10% η νέα εγγραφή να είναι ορφανό EXIT.
        - 45% να δημιουργείται τελείως νεα εγγραφή ENTRY.
        - 45% να δημιουργείται εγγραφή EXIT αντίστοιχη ενός ασθένή που υπάρχει
        ήδη στο σύστημα.

        Το τελευτάιο το καταφέρνω με έναν πίνακα που κρατάει τα στοιχεία των
        active cases πού έχουν εισαχθεί στο σύστημα.
